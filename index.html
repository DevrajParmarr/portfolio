NeuroAgent — Autonomous Software Engineering Agent
Most engineers use AI as autocomplete. NeuroAgent is different — a fully autonomous multi-agent system that accepts plain English and delivers working, tested, committed code. Built on LangGraph's stateful graph with a ReAct loop: PlannerAgent decomposes tasks → CoderAgent implements with GPT-4/Claude → TestAgent writes and executes tests in a Docker sandbox → ReviewerAgent quality-gates and loops back on failures → DeployAgent commits to GitHub and fires CI/CD. Dual-layer memory: Redis for in-session context, PostgreSQL+pgvector for semantic long-term recall. React frontend streams each agent's live reasoning via WebSockets.

Route Optimizer — Fleet Logistics Intelligence Platform
// The Problem

The Heterogeneous Fleet CVRP (HFCVRP) is NP-Hard. Real logistics companies — Amazon, Zomato, Dunzo — lose millions on suboptimal routing. Most academic solutions exist only on paper. I built a full-stack platform that solves it live on real map data, for 5–100 destinations with 2–50 mixed-capacity vehicles.

🔬 Algorithmic Core

Three routing strategies: 2-opt local search (iterative swap), Clarke-Wright Savings (savings-matrix merging), and Greedy Nearest-Neighbor. Each exposes different quality vs. speed trade-offs. Run all three simultaneously and compare side-by-side on the same instance.

⚙️ Backend Engineering

Node.js/Express handles geospatial ingestion, vehicle constraint modeling, and async execution. OpenStreetMap API provides real road-network distances — not Euclidean guesses. MongoDB persists problem configs and solution histories for session comparison.

🗺️ Interactive Map UI

React + Leaflet.js — define problems directly on the map. Click to add waypoints, configure demand per stop, set vehicle capacities. Optimized routes render as animated multi-color polylines per vehicle. Analytics dashboard surfaces fuel estimates and algorithm comparisons reactively.

📊 Real-World Impact

Reduces estimated fuel consumption by up to 20% vs. naive routing on benchmark cases. Handles edge cases: overloaded vehicles, isolated stops, unserviceable demand — graceful fallbacks throughout.
